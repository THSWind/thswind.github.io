latex-beamer tips

有时候为了找到一个字符或者实现某种布局的latex表示方法会花好多时间查网站查资料。比较花时间，现将我遇到的和解决的一些操作总结如下：

1 推荐文档

  1. lshort-zh-cn中文版本.pdf
  2. beameruserguide.pdf
  3. tikzpgfmanual.pdf
  4. tcolorbox.pdf

与其网上找别人的解决方法，不如自己去查官方文档。（不过偶尔网上找也有奇效）

2 九九八十一难

  1. 仅在当前页面（frame）有效的布局

    { % all template changes are local to this group.
    \setbeamertemplate{navigation symbols}{}
    \begin{frame}{研一(下学期)学习安排与预期成果}%[plain]

    \begin{tikzpicture}[remember picture,overlay]
    \node[at=(current page.center)] {
    \includegraphics[width=\paperwidth]{plan.png}
    };
    \end{tikzpicture}
    \end{frame}
    }

  2. 数学计算符号（主要是字体的变化）

    \mathbb{大写字母} 空心大写字母

    \mathcal{大写字母} 特殊字体的字母（常见于数学符号比如N，这里N表示高斯分布）

  3. 固定汇报与作者及单位

    \author{reporter：THSWind\[3ex]
    XXXXXX institute}
    \date{2019年04月11日}

  4. 图片布局与大小

    \includegraphics[width=\paperwidth]{plan.png} 与1结合使用效果更好，同理，height=\paperheig….可根据图片本身是长度更长还是宽度更宽来决定使用width还是height。

  5. 分栏

    \begin{frame}{基于Mnist数据集的CNN模型优化}

    ​ \begin{columns}[c] % The “c” option specifies centered vertical alignment while the “t” option is used for top vertical alignment

    ​ \column{.45\textwidth} % Left column and width
    ​ 写些想写的
    ​ \column{.5\textwidth} % Right column and width
    ​ 做些想做的
    ​ \end{columns}
    \end{frame}

  6. beamer->tikzpicture->node布局

    \node[at=(current page.center)] 此句是完全居中（但经测试，对于树状结构它并非完全居中的，这种情况看下面一句）

    \node[yshift=-1.5cm,xshift=0cm] at (current page.center) （使得y轴沿着中心往下偏移1.5厘米），可自己调整偏移量。

    其中center可被替换为east， west, north, south, north west等等。

  7. 表格生成方式

    https://www.latex-tables.com/

    https://www.tablesgenerator.com/

    推荐利用这两个网站图形化的操作实现后复制到你的latex之中就搞定。

  8. 空格

    姓　　名，两个字中间就是两个空格字符。有时候 \quad 没有效果，它有奇效。

会持续更新…..

Ubuntu: double screen+settings

由于使用双屏习惯了，在我装了windows10和ubuntu16.04的双系统之后，切换到ubuntu时。总是被ubuntu的双屏以及基本设置所耽误时间。所以在折腾了几次之后，我决定将ubuntu16.04下最常见的三种问题及其解决方法总结下来：

1 双屏设置

  • 问题1（描述）：在装了双系统后，首次打开ubuntu，由于是双屏，常常会出现有一边屏幕亮（无任何图标，但可以用鼠标右键，左键），另一边不亮（但又不是完全黑）。

    • 解决过程：后来，我重启过程中发现，两边屏幕都提示无vga信号。我意识到不亮这边的屏幕并不是因为坏了。为了验证此想法，我切换到window系统，开机后双屏正常扩展。说明屏幕没有坏的判断正确。

      既然没有坏，我想到另一种可能，通常双屏扩展时，通常图标都在其中一个屏幕上，另一个屏幕是空的。会不会现在亮着的这个屏幕就是空的那个。如果是的话。我只需要设置一下显示器就好了。

      问题来了，现在我只看得见鼠标和一个空屏幕，怎么做呢？

      1. 鼠标右键->更换背景 进入了设置，进入设备。显示那里，将当前显示器设置为主显示器。点一下 应用 搞定。如果未成功的话，恢复一下即可。（有的是因为英伟达的驱动bug导致）

    • 命令行解决：利用xrandr命令

      首先在终端输入：xrandr
      会显示目前的外置屏幕接口连接情况。我们注意哪些屏幕是连接的，一般有 dp, dvi,hdmi等等，与显示器型号以及所用的接口有关。以及primary（带有此单词的是主显示器）

      比如说，显示的信息中有：DP-1和HDMI-0是 connected。

      将DP-1设置为主显示器，命令如下：

      xrandr --output DP-1 --auto --primary

      将HDMI-0设置在 主显示器DP-1的右边，命令如下：

      xrandr --output HDMI-0 --right-of DP-1 --auto

2 输入法问题

默认安装的ubuntu16.04是没有中文输入法的，这点很不方便。所以我决定安装一个中文输入法，google拼音，搜狗输入法，选哪个呢，我选了第一个。搜狗太多广告了，花里胡哨的。只要打字速度跟得上，输入法什么的没多大关系。好，开始安装google拼音。

首先ubuntu下输入法的框架有iBus和fcitx。

  • 安装fcitx

    sudo apt-get install fcitx

  • 安装google拼音

    sudo apt-get install fcitx-googlepinyin

  • 启用fcitx

  • 进入setting->language,点一下manage部分,出现Language Support。

    在 Keyboard input method system处，选择 fcitx。

    记住，务必重启电脑。（有时候注销也可以）

  • 启用google拼音

    终端输入：fcitx-configtool

    弹出名为：Input Method Configuration的窗口

    点击窗口左下方的+号，取消勾选Only Show Current Language

    在下方搜索框输入：google pinyin

    在上方选中：Google Pinyin，通过左下方的上下符号将它置顶，点击ok。

这样google拼音安装完成。

3 禁止休眠问题

我们在跑机器学习相关的程序时，跑几天的代码是常有的事，可ubuntu老是会自己进入休眠状态，甚至无法被唤醒。这点十分不好。

  1. 进入setting， power，blank那里设置为 never

  2. 终端输入：setterm -blank 0

    注意：2 需要在root权限下输入。

Algorithm: SumTree practice

代码

代码来源于网络，我主要对这段代码进行了解读。并对代码最后6行用于测试的例子的代码计算过程做了详细的梳理。

import numpy as np

class Tree(object):
write = 0 # 表示第几个叶子节点可用（未被写入数据或可覆盖原数据）
def __init__(self, capacity):
self.capacity = capacity # capacity是叶子节点个数，
self.tree = np.zeros(2 * capacity) # 从1开始编号[1,capacity]
self.data = np.zeros(capacity+1, dtype=object) # 存叶子节点对应的数据data[叶子节点编号id] = data

def add(self, p, data):
    idx = self.write + self.capacity  
    self.data[self.write+1] = data
    self._updatetree(idx, p)
    self.write += 1
    if self.write > self.capacity:  
        self.write = 0
 
def _updatetree(self, idx, p):
    change = p - self.tree[idx]  
    self._propagate(idx, change)  
    self.tree[idx] = p  
 
def _propagate(self, idx, change):
    parent = idx // 2  
    self.tree[parent] += change  # 更新父节点的值，是向上传播的体现
    if parent != 1:
        self._propagate(parent, change)  
 
def _total(self):
    return self.tree[1]  
 
def get(self, s):
    idx = self._retrieve(1, s)  
    index_data = idx - self.capacity + 1  
    return (idx, self.tree[idx], self.data[index_data])
 
def _retrieve(self, idx, s):
    left = 2 * idx  
    right = left + 1
    if left >= (len(self.tree)-1):  
        return idx
    if s <= self.tree[left]:
        return self._retrieve(left, s)  # 往左孩子处查找
    else:
        return self._retrieve(right, s - self.tree[left])  # 往右孩子处查找

Parameter Space Noise For Exploration

这篇论文于2018年1月31日发表于ICLR会议。

1 简介

深度强化学习方法在探索行为方面通常通过在动作空间中加入噪声。而另一种可选的方法则是通过直接在agent的参数中添加噪声。诸如进化策略之类的方法用到参数干扰，但会丢弃过程中所有的时间结构，且需要更多的样本。将参数噪声和传统RL方法结合，可以将两者的优点结合起来。这篇论文证明了 off-policy 和 on-policy 方法都从这种方法中得到了改进，实验对比的算法有： DQN, DDPG, 和TRPO。实验环境在高维离散动作环境，且为连续控制任务。

探索依然是现代深度强化学习中的一个重要挑战，探索的主要目的是确保 agent 的动作不会提前收敛到一个局部最优解。

2 Off-Policy Methods

2.1 Deep Q-Networks(DQN)

DQN使用深度神经网络作为函数近似器去估计最优Q值函数，该等式符合Bellman最优方程。

2.2 Deep Deterministic Policy Gradients(DDPG)

DDPG是一种 actor-critic 算法，适用于连续动作空间。和DQN相似，其 critic 使用 off-policy 数据和递归的Bellman最优方程去估计Q值函数。

Perturbing deep neural networks

通过使用球形高斯噪声这种有意义的方法去干预深度神经网络的参数不是很明显，因为深度神经网络一般有数百万的参数和复杂的非线性交互。然而，2017年 Salimans等人提出了一种简单的网络再参数化将其实现。更确切的说，我们在被干预的层之间使用 layer normalization。由于这在同一层的不同激活之间normalizing，所以，即使不同的层对噪声的敏感度不同，也可以在所有层上使用相同的干扰尺度。

Adaptive noise scaling

参数空间噪声要求我们取一个合适的标量 sigma, 这可能是有问题的，因为该标量将强烈的依赖于特定的网络架构，并可能随着时间的推移而变化，因为随着学习的进展，参数对噪声变得更加敏感。另外，虽然容易直观的掌握动作空间噪声的尺度，但是很难理解参数空间的尺度。

我们提出了一个简单的方案，以一种简单而直接的方式解决上述所有的限制。这是通过随着时间来调整参数空间噪声的尺度。并将其与它引起的动作空间变化相关联来实现的。更具体地说，我们可以在动作空间中被干扰与非干扰策略之间定义一个距离度量。并根据它是否低于或者高于某个阈值来相应增加或者减少参数空间噪声。

Big V: vscode+anaconda

1 VScode的优点

VScode用于编写python程序再适合不过了。实在是居家必备、码首是瞻的神器。

  • 免费、开源、多平台支持（windows、Linux、mac）
  • 占系统资源少，内存上相对eclipse，vscode占用内存更少。eclipse约占478MB（1个进程），vscode约占200MB（6个进程）
  • 提供丰富的插件，左侧的工具列表，安装卸载十分方便。
  • 界面美观，功能强大。
  • 下方的terminal窗口方便对缺少的包进行安装。

2 Anaconda的优点

Anaconda则是新手入门的神器，免去了频繁安装各种包的痛苦，安装Anaconda之后，将其根目录，脚本，和bin目录添加到系统环境变量中。使用起来更加方便。

  • Anaconda附带了一大批常用的数据科学包（conda,python等180多个科学包及其依赖项），不需要单独下载。对新手比较友好。

  • conda，包管理器，十分强大。

两个强大的武器联手又会产生怎样绝妙的烟火呢？

3 Vscode+Anaconda

强强联手后，你将在vscode中享受两种利器的优点。特别指出的是，除了vscode免去安装某些包之外，还多了一个虚拟环境的功能。随时切换python编译环境。

在vscode左下角，底色为紫色的白色字体的python版本环境说明处（颜色可能因个人界面设计而异），点击它，可以看见vscode界面正上方出现一个下拉选项。出现多个python环境。和Anaconda的环境，包括Anaconda创建的虚拟环境。

以Vscode需要python2.7环境为例

  1. 打开Anaconda prompt程序
  2. 输入conda create -n env_name python=2.7
  3. 这样就创建一个名为env_name的python2.7的虚拟环境
  4. 再回到vscode中，点击左下角python版本那里
  5. vscode界面正上方出现一个下拉选项，选择我们刚创建的env_name虚拟环境即可。

Browser-related problems and solutions

1 google 浏览器内存不足

使用一段时间，偶尔会提示浏览器内存不足，通常是由于打开的页面过多，google占内存是出了名的。每个页面都有一系列子程序。

解决办法：

直接在浏览器输入：chrome://flags/#automatic-tab-discarding

把automatic tab discarding 那里改为 Enabled，重启浏览器即可。
